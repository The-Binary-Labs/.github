# The Binary Labs

We build in the open.

The Binary Labs is an open-source organization for experiments, tools, documentation, and maintainable building blocks. We use this space to ship ideas in public, improve them collaboratively, and keep contributor experience consistent across repositories.

## What you'll find here

- open-source repositories
- shared defaults for contribution and security
- focused experiments and reusable building blocks
- documentation and contributor guidance
- public issues and pull requests open to community participation

## Why this organization exists

We want new repositories to feel approachable from day one.

That means:

- clear contribution expectations
- predictable issue and pull request workflows
- respectful collaboration standards
- private handling for security reports and public discussion for normal project work

## How to contribute

Community contributions are welcome.

- Open an issue if you found a bug, want to suggest an improvement, or want to improve documentation.
- Open a pull request if you already have a focused fix or enhancement.
- For larger changes, start with an issue so we can align early.

You can find our shared community guidelines here:

- [Contributing Guide](https://github.com/The-Binary-Labs/.github/blob/main/.github/CONTRIBUTING.md)
- [Code of Conduct](https://github.com/The-Binary-Labs/.github/blob/main/.github/CODE_OF_CONDUCT.md)
- [Security Policy](https://github.com/The-Binary-Labs/.github/blob/main/.github/SECURITY.md)

## A note on issues

Normal issues are public and encouraged.

Please use public issues for bugs, feature ideas, and documentation requests. Only security vulnerabilities should be reported privately.
